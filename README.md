## 🚀 GitHub Actions Workflow Configuration

The automated pipeline is defined in [test-and-report.yml](file:///c:/Users/urmil/OneDrive/Dokumen/pdd/.github/workflows/test-and-report.yml) and runs on every `push` or `pull_request` to the `main` or `master` branches, as well as on manual `workflow_dispatch` triggers.

### Workflow Jobs:
1. **`frontend-selenium-tests`**: Runs Chrome Headless to test the frontend web application (300 E2E cases).
2. **`backend-api-tests`**: Spins up the Express server and executes REST API assertions (300 E2E cases).
3. **`appium-mobile-dryrun`**: Executes the mobile suite dry-run validation (300 mobile cases).
4. **`aggregate-test-reports`**: Gathers the separate Excel reports generated by each job, parses the statuses, generates a Markdown summary inside the GitHub Actions step summary, and uploads a single unified reports zip bundle.

## 🧪 Local Execution Instructions

To run the test suites locally, execute the following commands:

### 1. Backend REST API Tests
```bash
# Install dependencies and execute 300 API test cases
npm --prefix backend install
npm --prefix backend-tests install
npm --prefix backend-tests test
```

### 2. Frontend Selenium Web E2E Tests
```bash
# Install dependencies and execute 300 web test cases (requires Google Chrome)
npm --prefix selenium-tests install
npm --prefix selenium-tests test
```

### 3. Mobile Appium Dry-Run Tests
```bash
# Run dry-run validation of the 300 mobile test cases
npm --prefix appium-tests install
npm --prefix appium-tests run dry-run
```

## 📈 Aggregated Test Reports

After completion, each test runner generates a color-coded premium Excel report (`.xlsx`) detailing:
- **Test Case ID** (TC001 - TC100)
- **Category** (Functional, UI/UX, Performance & Security)
- **Description & Expected Outcome**
- **Execution Status** (`PASS`/`FAIL`)
- **Latency / Response Time** in milliseconds
- **Execution Timestamp**

These files are uploaded as run artifacts (`spendsense-all-reports`) in the GitHub Actions tab.
